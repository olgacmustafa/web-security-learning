# Labs Index

This directory contains the hands-on labs documented in this repository.

Each lab focuses on a specific web security concept and includes analysis from both technical and defensive perspectives.

---

## Available Labs

| Lab | Topic | Status |
|---|---|---|
| [Brute Force](./brute-force/) | Authentication attacks, password guessing logic, rate limiting awareness | Completed |
| [SQL Injection](./sql-injection/) | SQL injection basics, input manipulation, query abuse, mitigation thinking | Completed |
| [File Upload](./file-upload/) | Unsafe file upload logic, validation weaknesses, upload filtering concepts | Completed |
| [SSTI](./leaf-ssti/) | Server-Side Template Injection, template engine abuse, secure rendering awareness | Completed |
| [LFI](./venomous-lfi/) | Local File Inclusion, path traversal logic, file access abuse, secure file handling | Completed |

---

## Lab Writing Standard

Each lab may include some or all of the following files:

- `overview.md`
- `methodology.md`
- `exploitation.md`
- `mitigation.md`
- `results.md`
- `lessons-learned.md`
- `conclusion.md`

---

## Purpose

The purpose of these labs is to:

- document my cybersecurity learning journey
- practice vulnerability analysis in authorized environments
- build a structured public portfolio in web security
- improve both offensive understanding and defensive thinking
